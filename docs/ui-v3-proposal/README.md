## 一、整体设计思路

### 1.1 设计语言：「Studio Blue · 深空工作室」

把 SmartSub 从「文档型工具页面」升级为「媒体工作室软件」。核心心智模型改变：

- 现状：每页是一篇「带表单的文章」——大标题、描述、居中容器、纵向滚动。
- 目标：每页是一个「工作室面板组合」——外壳（导航/顶栏/状态栏）恒定，内容区由 2~4 块功能面板拼装，信息在面板内部密排。

四条基本法：

1. **面板化（Panel-first）**：内容区绝不裸排内容。任何信息都装进带边框、带标题行的面板（panel）里；面板标题行 38px，内容紧贴排布。
2. **四级视觉层级**：`chrome 外壳 → canvas 画布 → panel 面板 → element 元素`，用背景色阶 + 1px 边框区分，浅色再加一层极轻阴影；不靠大间距和分割线堆层次。
3. **密度对齐 NLE**：基准字号 13px（辅助 11~12px），控件高 28px（主按钮 32px），列表行高 40~42px，面板间距 10px，页面四周留白 12px。信息密度对齐剪映专业版/达芬奇，而不是营销网页。
4. **状态常显**：新增底部全宽状态栏（26px），常驻引擎状态、GPU 后端、当前模型、翻译服务、任务队列、版本号——专业工具的「仪表盘感」主要来自这里。

### 1.2 配色体系

主色保持蓝色系（P0 约束），在现有 shadcn HSL 变量体系上扩展，不引入新方案。

**品牌主色（Editing Blue 收敛为 Studio Blue）**

| 语义          | 浅色        | 深色         | 说明                                         |
| ------------- | ----------- | ------------ | -------------------------------------------- |
| primary       | `#1668dc`   | `#409cff`    | 浅色加深保证白字 AA；深色提亮保证深底可读    |
| primary-hover | `#2a78e8`   | `#5facff`    |                                              |
| primary-soft  | 9% 透明主色 | 13% 透明主色 | 选中底、推荐条、hero 卡                      |
| on-primary    | `#ffffff`   | `#04243e`    | 深色模式实心按钮用深蓝黑前景（延续现有做法） |

**功能色（每色配 soft 底色版，用于徽章/横幅）**

| 语义    | 浅色      | 深色      |
| ------- | --------- | --------- |
| success | `#15963f` | `#3dd68c` |
| warning | `#d97706` | `#f5a623` |
| error   | `#dc2626` | `#f87168` |
| info    | `#0284c7` | `#57b6ff` |

**中性阶（关键改动：深色由纯灰 0% 饱和改为微蓝冷灰，解决"发灰发脏"）**

| 层级                   | 浅色                              | 深色                              | 对应现有变量                                 |
| ---------------------- | --------------------------------- | --------------------------------- | -------------------------------------------- |
| canvas 画布            | `#f6f7f9`                         | `#101114`                         | `--background`                               |
| chrome 外壳            | `#eef0f3`                         | `#16181c`                         | `--chrome`                                   |
| panel 面板             | `#ffffff`                         | `#1b1e23`                         | `--card`                                     |
| panel-2 面板内嵌面     | `#f8f9fb`                         | `#22262c`                         | 新增 `--panel-2`                             |
| hover / active         | `#f0f2f5` / `#e9edf2`             | `#262b32` / `#2c323a`             | `--accent` 拆分                              |
| border / border-strong | `#e3e7ed` / `#cdd5df`             | 白 7% / 白 14%                    | `--border` 拆两档                            |
| 文字 t1 / t2 / t3      | `#17202b` / `#4e5a69` / `#8b95a5` | `#e9ecf1` / `#a3adbb` / `#667080` | `--foreground` / `--muted-foreground` 拆三级 |

要点：

- 浅色画布从近白 99% 压到 `#f6f7f9`，让白色面板「浮」出来（现状白纸上放白卡，层次靠边框硬撑）。
- 深色面板层从 11% 灰提到 `#1b1e23`（约 12% 带蓝相），边框改半透明白，避免深色下边框忽明忽暗。
- 浅色阴影只用两档极轻（panel: `0 1px 2px 4%`；raised: 浮层用）；深色不用投影表层级，改用背景抬升 + 顶部 1px 内高光。

### 1.3 布局原则

1. **外壳固定三件套**：左侧 64px 竖排导航 rail（图标上/文字下）+ 顶部 44px 工具栏（面包屑 · ⌘K 搜索 · 加速徽章/通知/帮助/主题）+ 底部 26px 状态栏。顶栏从 57px 压到 44px，净赚 13px 内容高度。
2. **导航分组**（P0 约束 3）：任务组（启动台/字幕/校对/合成/配音）与配置组（引擎/翻译/声音）之间加分隔线，设置固定沉底。选中态 = 主色 soft 底 + 左缘 3px 指示条 + 图标文字同色。
3. **页面按属性分四种版式，不套同一模板**：
   - 启动台（home）：`任务卡 + 最近任务表格 ｜ 右侧栏（拖放/就绪度/继续工作）` 双栏仪表盘
   - 编辑器页（合成/配音/校对编辑）：`左 inspector 332px ｜ 右舞台（预览+走带+时间轴）｜ 底部行动条` NLE 三区
   - 批处理页（tasks/[type]）：`顶部配置条 ｜ 文件队列表格 ｜ 底部行动条` 队列式
   - 配置页（引擎/翻译/声音）：`左主从列表 252px（含总览首项）｜ 右详情面板` 主从式；设置页为单列分区表单
4. **视觉热区**：主行动按钮（生成/开始/导出）永远在右下角行动条；环境就绪度、推荐操作放右上侧栏；次要配置一律折叠后置。
5. **交互动线统一**（P0 约束 7），三条硬规则：
   - 所有配置页（引擎/翻译/声音）左栏第一项都是「总览」：就绪状态 + 推荐起步路径（现在只有配音服务有）。
   - 所有任务页空态都用同一个「三步引导」组件（1-2-3 步 + 主按钮 + 拖放提示）：配音页已有此形态，推广到合成、校对、批处理页，替代校对页现在的「整页三卡导入」。
   - 头部两种模式明确分工：枢纽页用面包屑 + 面板标题；工作页用「返回 + 工程名」面包屑（顶栏内完成，不再页内另起一行大标题）。

### 1.4 技术落地映射（不改功能，红线内）

- token 全部落在 `renderer/styles/globals.css` 的 CSS 变量 + `renderer/tailwind.config.js` 扩展色（`--panel-2`、`--border-strong`、`--hover`、文字三级等新增变量，沿用 HSL/hex 变量模式），双主题同 key 双值。
- 组件重做只动 `renderer/components/ui/*` 的 variants 与尺寸（button 28/32px、input 28px、select 28px 等），调用方类名尽量不动。
- 外壳改造集中在 `renderer/components/Layout.tsx`（rail 竖排 + 底部状态栏）；状态栏数据源复用现有 IPC（GPU 徽章、任务轮询、模型下载已有）。
- 不新增第三方 UI 库；lucide 图标继续用。

## 二、核心页面改版方案（3 个）

### 2.1 启动台 `/home`

**原问题**

- 单列 `max-w-4xl` 居中，1440px 窗口下左右各浪费 ~250px；任务卡下方大片空白。
- 环境信息只有一条黄色警示横幅，就绪时看不到任何环境状态；GPU/模型/翻译服务状态分散在顶栏角落。
- 「最近任务」只是列表预览，无状态列、无操作列；空态占据巨大面积。
- 用户上次干到一半的工作没有任何「接续」入口。

**调整后布局**

```
┌──────────────────────────────┬──────────────┐
│ 开始新任务（6 卡，3×2 紧凑）    │ 拖放快速开始    │
│  卡=图标+标题+一句话+拖放槽    ├──────────────┤
├──────────────────────────────┤ 环境就绪度      │
│ 最近任务（表格化）             │  模型/GPU/翻译  │
│  文件名+路径 类型 时间 状态 操作│  /配音 4 行     │
│  行高 40px，含进行中/失败徽章  ├──────────────┤
│                              │ 继续上次的工作   │
│                              │  流水线进度+按钮 │
└──────────────────────────────┴──────────────┘
```

**优化理由**

- 双栏填满宽度，信息密度翻倍；任务卡从「装饰卡」变「入口 + 拖放热区」，保留原拖放逻辑。
- 「环境就绪度」把分散状态聚合成 4 行仪表（也是引导入口：未配置项直接给「去配置」），替代只在出错时出现的警示横幅——就绪与否都可见。
- 最近任务表格化后承担「任务队列」职责，进行中/失败状态一眼可扫，配合状态栏的队列指示。
- 「继续上次的工作」显性化了产品固有的流水线概念（转写→翻译→校对→合成），是纯信息组织，不加新功能。

### 2.2 合成到视频 `/subtitleMerge`

**原问题**

- 顶部两个通栏文件选择框占一整行，选完后仍占大量空间；预览黑框偏小且埋在右栏中部。
- 左栏样式设置从上滚到下（预设/字体/颜色/位置/效果/高级），无分区标题层级，找项靠滚。
- 输出方式/画质/路径/生成按钮挤在右下一张卡里，主行动按钮不在视觉热区；「生成视频」是全宽大蓝条，反而廉价。
- 整页没有时间维度，无法感知字幕在片中的分布。

**调整后布局（NLE 三区）**

```
┌────────────┬───────────────────────────────┐
│ 字幕样式     │ 文件条：视频槽│字幕槽│烧录/软封装 │
│ (inspector) ├───────────────────────────────┤
│  预设(缩略图)│                               │
│  字体       │        视频预览舞台             │
│  位置与效果  │     （字幕实时渲染+安全区）      │
│  双语排布    ├───────────────────────────────┤
│  高级(折叠)  │ 走带：时间码 ◀ ▶ 播放 音量 全屏  │
│            ├───────────────────────────────┤
│            │ 迷你时间轴：V1 视频轨 / S1 字幕块 │
│            ├───────────────────────────────┤
│            │ 输出条：画质·输出路径·预估│生成视频│
└────────────┴───────────────────────────────┘
```

**优化理由**

- 与剪映/达芬奇的「左属性-右画布-底导出」范式对齐，专业用户零学习成本；预览成为页面主角（占 ~55% 面积），样式调整立即所见即所得。
- 文件选择压成 30px 文件条（选中态绿点 + 元信息），把两条通栏的空间还给预览；输出方式（烧录/软封装）作为分段控件放同一条，属于「输入决策」而非「输出配置」。
- 左栏用大写小标题分区（预设/字体/位置与效果/双语排布），行内 label+控件两栏对齐，扫描效率高；高级选项折叠沉底。
- 迷你时间轴展示字幕块分布（数据来自已解析的字幕条目，只读展示），点击可 seek——低成本获得「时间维度」感知。
- 底部输出条 = 全局行动条：路径、预估大小、生成按钮固定右下热区。

### 2.3 引擎与模型 `/engines`（配置页范式代表）

**原问题**

- 左栏直接落在 whisper.cpp 详情，无总览——与配音服务页（有总览）动线不一致。
- 右栏是长滚动流水账：GPU 大横幅（浅绿大块）+ 推荐横幅 + 搜索 + 模型分档列表，层级平、推荐信息淹没。
- 模型列表每行信息（速度点/精度点/大小/下载）排版松散，分档标题与行混在一起。

**调整后布局**

```
┌───────────────┬─────────────────────────────────┐
│ 听写引擎        │ 引擎头：图标+名称+徽章│设为默认 ⋯ │
│  ◻ 总览        ├─────────────────────────────────┤
│  — 本地引擎 —   │ ▸ GPU 加速 strip（1行：状态+模式段控│
│  ● whisper.cpp │ ▸ 推荐 strip（1行：small+一键下载） │
│  ○ faster-wh…  ├─────────────────────────────────┤
│  ○ 多模型引擎    │ 搜索│全部/已安装│模型目录│导入      │
│  — 云端听写 —   ├─────────────────────────────────┤
│  ○ OpenAI      │ 模型表格：名称│说明│速度/精度│大小│操作│
│  ○ Groq …      │  （分档做行内分组条，推荐行高亮）     │
└───────────────┴─────────────────────────────────┘
```

**优化理由**

- 左栏加「总览」首项 + 分组标题（本地引擎/云端听写）+ 状态点，与翻译服务、配音服务共用同一套主从模板——三个配置页动线完全一致（P0 约束 7）。
- GPU 与推荐从「大横幅」压成两条 28px strip：状态色图标 + 一句话 + 行尾操作，信息不减、高度减半；GPU 模式（自动/仅GPU/仅CPU）用分段控件行内直达，不用再进折叠区。
- 模型库表格化：等宽字体模型名、速度/精度改为紧凑双 gauge、分档变行内分组条，推荐行主色高亮底——列表可扫描性大幅提升，同样高度多显示 ~40% 行数。

| 5   | 启动台「继续上次的工作」 | **保留**（从最近任务派生最新未完结任务 + 流水线阶段，无新存储）；**2026-07-16 反转：移除**——recipes-launchpad 落地后最近任务行自身已支持回开续跑（向导任务页按 configSnapshot 恢复、配音会话可恢复），派生卡片与列表首行完全重复；右栏空间改为「工具箱」常显面板（校对/合成/配音单文件工具入口），窄屏由左栏工具行兜底 |
| 6   | 落地节奏                 | **三批 PR**：① token + 外壳（rail/顶栏/状态栏/基础组件）→ ② 三个核心页（启动台/合成/引擎）→ ③ 其余页面（任务页/校对/配音/翻译/声音/设置）+ 弹窗浮层统一                                                                                                                                                                |

**遗留事项**（落地阶段处理）：

- 圆角基准：demo 用面板 10px / 控件 7px，落地时以 `--radius` 变量实现，视觉验收时可整体微调
- Windows 适配：本机为 macOS，Windows 下的标题栏与字体渲染需在落地阶段用 Windows 环境或 CI 截图验证
