## ADDED Requirements

### Requirement: 工作台克隆音色集成

工作台引擎下拉 SHALL 纳入 zipvoice 克隆引擎（就绪 = 模型已安装），其音色列表 SHALL 为用户的「我的音色」清单（zipvoice 引擎无内置音色）；选中该引擎且无任何克隆音色时 MUST 展示「创建克隆音色」引导入口（打开创建向导，完成后音色列表即时刷新）。克隆音色在行级 voice 覆盖、试听、单行重生成等既有交互中 MUST 与内置音色行为一致。

#### Scenario: 克隆音色出现在下拉

- **WHEN** 用户创建了克隆音色并在工作台选择 zipvoice 引擎
- **THEN** 音色下拉列出全部我的音色（按名称展示），选中后可试听与批量合成

#### Scenario: 空态引导创建

- **WHEN** zipvoice 模型已安装但用户尚无克隆音色
- **THEN** 配置栏出现创建引导按钮，完成向导后无需刷新页面即可选中新音色

## MODIFIED Requirements

### Requirement: 配音服务独立页面

系统 SHALL 提供独立「配音服务」导航页(形制「引擎与模型」的主从双栏),统一管理:本地 TTS 模型(每个模型一个左栏条目:下载/进度/删除/导入/打开目录)、在线配音服务商(**每个服务商一个平级左栏条目逐条外显**——OpenAI、硅基流动、Edge TTS 等,新服务商可扩展追加)与**我的音色**(每个克隆音色一个左栏条目,组内含创建入口与空态引导);服务商条目 = 配置表单 + 测试连接 + 已配置状态点(形制同云 ASR 的 CloudProviderPanel),协议型(OpenAI 兼容)MUST 支持「添加自定义」接入任意兼容端点(多实例)。

#### Scenario: 页面入口与条目外显

- **WHEN** 用户打开「配音服务」页
- **THEN** 左栏分「本地模型」(kokoro、vits-zh、zipvoice 逐条)、「在线服务」(OpenAI、硅基流动、Edge TTS 逐条 + 添加自定义)与「我的音色」(克隆音色逐条 + 创建入口)三组,选中任一条目右栏即为其管理面板

#### Scenario: 自定义端点扩展

- **WHEN** 用户点击「添加自定义」并命名新实例
- **THEN** 生成新的 OpenAI 兼容实例条目,可独立配置 Base URL / Key / 音色并测试连接
