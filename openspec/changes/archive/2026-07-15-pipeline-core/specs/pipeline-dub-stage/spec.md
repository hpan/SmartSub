# pipeline-dub-stage Spec Delta

## ADDED Requirements

### Requirement: 任务级配音附加阶段

任务 SHALL 支持可选配音附加阶段（`formData.dub`，与工作台 DubbingConfig 同构：引擎/音色/整体语速/克隆质量/本地并行路数/过长兜底/重叠模式）：字幕生产段完成后逐文件执行批量配音，阶段状态以既有字段名约定挂在文件上（`dubbing`: `''|loading|done|error` + `dubbingProgress`/`dubbingError`），事件镜像与工作项持久化沿用既有通道。未配置附加阶段的任务行为 MUST 与现状完全一致。

#### Scenario: 一条龙任务执行配音阶段

- **WHEN** 一个「生成并翻译 + 配音」任务的某文件完成翻译
- **THEN** 该文件进入配音阶段（dubbing=loading，进度按行数推进），完成后 dubbing=done，任务列表实时可见

#### Scenario: 无附加阶段任务不受影响

- **WHEN** 用户运行既有三类任务（未配置 dub/compose）
- **THEN** 执行流程、事件与产物与引入附加阶段前完全一致

### Requirement: 配音文本源为纯译文优先

配音阶段的文本源 SHALL 按优先级解析：带翻译的任务用纯译文（纯译文中间产物；仅存在双语交付物时经校对 sidecar 的译文列重建纯译文），无翻译任务用源字幕。MUST NOT 将双语字幕直接送入合成（避免同行双语被朗读两遍）。

#### Scenario: 双语交付物重建纯译文

- **WHEN** 任务译文交付物为「原文+译文」双语格式且配音阶段开启
- **THEN** 配音以重建的纯译文文本执行，行时间轴与交付字幕一致

### Requirement: headless 会话执行与产物

配音阶段 SHALL 以 headless 配音会话执行（复用会话持久化：`file.dubbingSessionId` 记录关联），产物为完整配音轨（wav，锚定媒体时间轴）与可选顺延版字幕，路径记录在文件字段（`dubbedTrackPath`/`shiftedSubtitlePath`）供合成阶段与完成后操作消费；MUST NOT 在配音阶段做视频封装（统一交给合成阶段）。

#### Scenario: 产物可被下游消费

- **WHEN** 某文件配音阶段完成
- **THEN** 配音轨 wav 与（时移发生时的）顺延字幕落盘，文件字段携带路径；后续合成阶段与工作台回开均可直接使用

#### Scenario: 工作台可回开检视

- **WHEN** 用户从任务详情打开某文件的配音会话（或经最近任务）
- **THEN** 配音工作台按 sessionId 恢复行级状态，可试听、单行重生成

### Requirement: 全自动兜底与失败语义

配音阶段 MUST 全自动完成不等待人工：过长行按 overflow 配置（截断/顺延，默认截断）在拼接期自动处置；单行失败不中断批量。批量结束仍存在失败行时该文件阶段 SHALL 判定为 error（不进入合成阶段），重试 MUST 仅重跑失败/未完成行（已完成行经会话持久化跳过）。

#### Scenario: 过长行自动处置

- **WHEN** 全自动任务中某行所需倍率超红线
- **THEN** 该行按配置截断或顺延进入配音轨，任务不停等人工确认

#### Scenario: 失败重试续跑

- **WHEN** 某文件配音阶段因 3 行云端合成失败而 error，用户点击重试
- **THEN** 仅重跑失败行，成功后阶段转 done 并继续后续阶段

### Requirement: 跨文件互斥与取消

配音阶段 SHALL 过全局互斥闸：同一时刻仅一个文件执行配音（行级并发按引擎配置），其余文件的听写/翻译阶段照常并行，排队文件的配音阶段呈现等待中。任务取消 MUST 中止当前配音会话（在途行中止、半成品清理），排队文件不再进入配音。

#### Scenario: 批量任务的配音串行

- **WHEN** 并发 3 的任务中两个文件同时到达配音阶段
- **THEN** 一个执行、一个排队等待，第三个文件的转写/翻译不受影响

#### Scenario: 取消联动会话中止

- **WHEN** 用户取消任务时某文件配音进行中
- **THEN** 该会话批量中止（在途行回退待合成），阶段状态按取消语义回退
