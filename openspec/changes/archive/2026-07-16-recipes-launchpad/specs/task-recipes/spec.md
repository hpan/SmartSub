# task-recipes Spec Delta

## ADDED Requirements

### Requirement: 配方模型与持久化

系统 SHALL 支持任务配方：命名的「目标勾选（翻译/配音/成品视频）+ 输入类型 + 任务配置快照（字幕段配置与 dub/compose/gates）」。用户配方 SHALL 持久化于应用存储并提供增/删/改（重命名）查询接口；内置配方以代码常量提供（不落存储、不可删除、文案随 i18n）。

#### Scenario: 用户配方持久化

- **WHEN** 用户保存一个名为「课程搬运」的配方后重启应用
- **THEN** 配方仍在，包含保存时的目标勾选与全部阶段配置

#### Scenario: 内置配方不可删除

- **WHEN** 用户查看内置配方
- **THEN** 无删除/重命名入口；应用升级后内置配方定义随版本更新且无需数据迁移

### Requirement: 内置配方清单

内置配方 SHALL 至少包含四张（白话命名）：「视频 → 配音成片」（完整流水线，含默认人工把关：字幕校对开、配音确认关，主推位）、「视频 → 双语字幕」、「视频 → 字幕」、「字幕翻译」——后三张与既有三类任务语义一一对应，承接旧启动台卡片的心智与深链习惯。

#### Scenario: 老任务类型无损承接

- **WHEN** 用户点击「视频 → 双语字幕」内置配方
- **THEN** 进入既有 generate-translate 任务页，功能与旧卡片完全一致

### Requirement: 配方应用规则

配方应用 SHALL 按内容分流：无附加阶段（不含配音/成品视频）的纯字幕配方直达对应既有任务页（含拖放即建工程直跑）；含附加阶段的配方打开新建任务向导并按配方预填（目标、字幕段配置、配音、合成、人工把关；拖放文件一并带入）。分流与预填推导 MUST 为可单测的纯函数。

#### Scenario: 一条龙配方进向导预填

- **WHEN** 用户往「视频 → 配音成片」配方卡拖入 3 个视频
- **THEN** 进入向导：文件已带入、四个目标预勾、把关按配方默认，确认后开始

#### Scenario: 配方配置容错

- **WHEN** 应用一个引用了已被删除的 TTS 服务商实例的用户配方
- **THEN** 向导正常打开，配音引擎按既有回落规则选择就绪项或呈现内联引导，不报错不阻塞
