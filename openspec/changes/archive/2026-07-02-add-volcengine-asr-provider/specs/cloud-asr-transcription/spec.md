## ADDED Requirements

### Requirement: 服务商类型可声明音频上传约束

云 ASR 服务商类型 SHALL 可以在其类型定义上声明可选的音频上传约束（最大上传字节数、单切片最长时长）。云引擎在准备/切片音频时 SHALL 采用所选实例类型声明的约束值；类型未声明某项时 SHALL 回落到全局默认值。既有未声明约束的类型行为 MUST 与变更前完全一致。

#### Scenario: 按类型声明的上限压缩与切片

- **WHEN** 所选云实例的类型声明了 `maxUploadBytes`，且本地音频体积超过该值
- **THEN** 云引擎按该声明值（而非全局默认）触发压缩，压缩后仍超限则按声明的切片时长切片转写

#### Scenario: 未声明约束回落全局默认

- **WHEN** 所选云实例的类型未声明任何音频上传约束（如既有 OpenAI 兼容 / ElevenLabs / Deepgram）
- **THEN** 云引擎沿用全局默认上限，压缩/切片行为与本变更之前一致

### Requirement: 火山引擎豆包服务商类型（录音文件识别极速版）

系统 SHALL 提供品牌型硬单例服务商类型 `volcengine`（火山引擎·豆包大模型录音文件识别极速版）：凭据为必填的单 API Key（新版「豆包语音」控制台「API Key 管理」签发；旧版控制台 App ID + Access Token 两件套不支持），模型清单默认 `bigmodel`，可选自定义 Base URL。鉴权 SHALL 使用单 `X-Api-Key` header。转写 SHALL 以 base64 直传本地音频至极速识别端点、同步获取结果，MUST NOT 要求用户提供公网音频 URL 或自备对象存储。该类型 SHALL 注册进类型分发表并支持连接自测；缺必填凭据时 SHALL 判为未配置且不可被任务选用。凭据提示 SHALL 说明火山方舟 / 大模型推理的 API Key 与豆包语音不通用、且需先开通极速版资源。

#### Scenario: 配置火山实例并被任务选用

- **WHEN** 用户在云端听写面板的火山引擎分区填入豆包语音 API Key（模型保持默认 `bigmodel`）
- **THEN** 该实例判为已配置，出现在任务页「引擎 ▸ 模型」下拉中，可选 `bigmodel` 发起转写

#### Scenario: base64 直传完成转写

- **WHEN** 任务以火山实例转写一个本地视频
- **THEN** 系统把抽取的本地音频以 base64 编码放入请求体直传极速识别端点，单次请求同步取回识别结果并写出 SRT，全程不要求公网 URL

#### Scenario: 凭据不全判为未配置

- **WHEN** 火山实例缺少 API Key
- **THEN** 该实例在面板置于「未配置」态，任务页下拉不将其列为可用转写来源

### Requirement: 模型清单录入结构化

云服务商实例的模型清单 SHALL 按类型定义结构化录入：类型声明单一固定模型时 UI SHALL 只读展示、MUST NOT 允许修改；类型声明可枚举模型集合时 UI SHALL 以点选方式启停、MUST NOT 提供自由文本录入；模型不可枚举的协议型（OpenAI 兼容）SHALL 以标签式录入（输入确认成标签），MUST NOT 要求用户手拼分隔符。模型清单解析 SHALL 宽容历史数据中的全角/半角逗号、顿号、分号与换行分隔。

#### Scenario: 固定模型只读

- **WHEN** 用户查看火山引擎实例的模型字段（固定 `bigmodel`）
- **THEN** 模型只读展示，无输入框、不可修改

#### Scenario: 枚举模型点选

- **WHEN** 用户配置 ElevenLabs（scribe_v2/scribe_v1）或 Deepgram（nova-2/nova-3）实例的模型
- **THEN** 以点选标签启停模型，无自由文本输入；历史存量中不在枚举内的模型 id 仍可见且可移除

#### Scenario: 自由清单标签式录入

- **WHEN** 用户为 OpenAI 兼容实例录入模型 id 并键入回车或任意分隔符（含全角逗号）
- **THEN** 输入即刻转为标签，存储为规范分隔的清单，混用全角/半角标点不产生脏数据

### Requirement: 火山转写结果的词级归一与标点回贴

火山转写 SHALL 请求分句与逐字时间戳（`show_utterances` 开启、标点开启），并把逐字时间戳（毫秒）归一为内部词级结果（秒）走既有词级成句管线；因火山逐字结果不含标点，系统 SHALL 复用既有标点回贴逻辑把整段文本的标点回贴到逐字序列。分句结果 SHALL 同时归一为段级结果作为降级兜底。系统 MUST NOT 为火山另造成句逻辑。

#### Scenario: 逐字时间戳成句并带标点

- **WHEN** 火山返回带 `utterances[].words[]` 逐字时间戳（无标点）与含标点的整段文本
- **THEN** 系统把逐字时间戳换算为秒、回贴标点后经既有成句管线切分为多条自然字幕，字幕文本含标点

#### Scenario: 缺词级结果时段级兜底

- **WHEN** 某次火山响应仅含分句（utterances）而无逐字 words
- **THEN** 系统以分句的起止时间生成段级字幕，任务仍成功产出 SRT

### Requirement: 火山响应以服务状态码判定成败与重试

火山转写 SHALL 以响应的服务状态码头（`X-Api-Status-Code`，`20000000` 为成功）判定成败，MUST NOT 仅凭 HTTP 状态码判定。静音音频状态码（`20000003`）SHALL 视为空结果成功（不产出字幕、不判失败）。参数类错误（参数无效/空音频/格式不正确）与鉴权类失败 SHALL 不重试并透出服务端错误信息；服务器过载/服务内部错误/HTTP 429/5xx/网络错误/超时 SHALL 按既有「显式超时 + 指数退避有限重试」策略处理；未知状态码 SHALL 以明确错误结束并携带服务端消息，MUST NOT 静默挂起。

#### Scenario: HTTP 200 但状态码为失败

- **WHEN** 火山响应 HTTP 200 但 `X-Api-Status-Code` 非成功值
- **THEN** 系统按该状态码语义处理：可重试类退避重试，不可重试类以携带服务端消息的错误结束任务

#### Scenario: 鉴权失败不重试

- **WHEN** 请求因 API Key 无效被拒（如误填火山方舟的 API Key 导致 `401 Invalid X-Api-Key`）
- **THEN** 系统不重试，直接以包含服务端原因的鉴权错误结束

#### Scenario: 连接自测区分凭据问题

- **WHEN** 用户对火山实例执行连接测试
- **THEN** 系统以最小合法音频（1 秒静音 WAV）发起真实探测——服务端参数校验先于鉴权，空音频探测无法验证凭据；有效凭据（返回成功或静音空结果）判为连通成功，无效凭据判为失败并展示服务端返回的原因
