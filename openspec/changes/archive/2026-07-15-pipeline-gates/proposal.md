# Proposal: 人工闸门（P3）——流水线的字幕校对与配音确认检查点

## Why

P2 已让「视频→字幕→翻译→配音→成片」一条龙全自动跑通，但全自动意味着字幕错漏会被直接烧进成片、云端配音在错误文本上白花钱。产品方向定稿的第三步：在流水线上提供**人工检查点**——字幕校对（进配音/合成前把关文本）与配音确认（进合成前把关声音），用户可选人工或全自动，文件级流式（到闸即可校对、逐个放行，不等整批）。界面用白话（「人工校对/人工确认」），不暴露"闸门"模型词。

## What Changes

- **闸门配置**：任务配置新增 `gates`（字幕校对 / 配音确认，各 `manual|auto`）；向导在勾选配音或成品视频时显示「人工把关」开关——字幕校对**默认开**（下游要花钱/花时间，停一下有真实价值），配音确认默认自动。
- **停靠语义（核心机制）**：文件到达人工检查点即置闸门字段为「待校对」并**结束本轮执行**（不占并发槽、随事件镜像天然持久化、跨重启保留）；下游阶段不执行。到闸发系统通知（任务级聚合节流）；有文件停靠时任务完成通知改为「有 N 个文件等待校对」。
- **放行语义**：新增放行 IPC——把闸门置「已通过」并按任务配置快照**重新派发**该文件（复用 P2 续跑：已完成段落直接跳过，从闸门后继续）；支持单文件与全部放行。校对若修改了字幕，配音文本源 hash 变化会自动重建配音会话（重配修改行）；配音确认中若重生成了行，放行后配音轨自动重建再合成。
- **任务详情闸门 UX**：轨道上渲染菱形检查点（待校对高亮）；顶部聚合操作条「N 个文件等待字幕校对/配音确认 [逐个校对][全部放行]」；字幕检查点行级「校对」打开内嵌校对编辑器（外壳提供「放行并继续下一个」流式审片），「放行」直接通过。
- **配音工作台检查员模式**：配音检查点的「检查配音」跳转工作台（按文件 sessionId 恢复行级状态），顶部检查员上下文条：任务/文件定位、上一个/下一个待检文件、「放行并继续」、「全部放行」、返回任务。
- 兼容：未配置 gates 的任务（含全部既有任务与 P2 已建任务）行为与现状完全一致。

不包含（P4）：配方保存、启动台改版、旧任务页收敛、字幕输入视频配对。

## Capabilities

### New Capabilities

- `pipeline-gates`: 闸门配置与默认值、停靠语义（状态字段/结束本轮/持久化/跨重启）、到闸通知与完成通知联动、放行语义（单文件/批量、快照续跑、校对后自动重配、确认后音轨重建）、取消与删除语义。
- `gate-review-ux`: 任务详情的检查点呈现（轨道菱形、聚合操作条、行级校对与放行）、字幕检查点的内嵌校对编辑器检查员包壳（放行并继续下一个）。

### Modified Capabilities

- `pipeline-task-wizard`: 新增「人工把关」开关要求（显示条件、默认值、白话文案）。
- `dubbing-workbench`: 新增「检查员模式」要求（流水线上下文条、放行并继续、待检文件导航、全部放行）。

## Impact

- **主进程**：`fileProcessor.ts`（两个闸门检查点 + dubbingGate 放行后音轨重建）、新增 `pipeline/gateManager.ts`（到闸通知聚合、放行派发）、`taskProcessor.ts`（导出主进程内部派发入口，复用 progressWindow 合成事件）、新增 IPC `pipeline:releaseGate`；`types/types.ts`（`IFormData.gates`、`IFiles.subtitleGate/dubbingGate`）。
- **渲染层**：`stageUtils`/`TaskRowList`/`TaskGridList`（菱形检查点渲染与行级动作）、任务页（聚合操作条 + 校对检查员包壳）、`DubbingPanel`/`useDubbing`（检查员模式）、向导（人工把关开关）、`locales/{zh,en}/{tasks,dubbing}.json`。
- **兼容性**：gates 缺省即全自动，既有任务零变化；闸门字段为增量可选字段，workItem 结构向后兼容；「待校对」状态不参与中断标记（跨重启保留）。
